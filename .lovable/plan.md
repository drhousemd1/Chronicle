

# Add Google & Apple Sign-In to Auth Modal

## What Changes

Add Google and Apple OAuth buttons to the `AuthModal` (and the `/auth` page) above the email/password form, separated by an "or" divider — matching the pattern in the reference screenshot.

## Prerequisites

1. **Configure Social Login** — need to run the "Configure Social Auth" tool first to generate the `src/integrations/lovable/` module and install `@lovable.dev/cloud-auth-js`. Both Google and Apple are managed by Lovable Cloud with no API keys needed.

## Implementation

### File: `src/components/auth/AuthModal.tsx`
- Import `lovable` from `@/integrations/lovable/index`
- Add two OAuth buttons above the email form:
  - **Google** button with Google "G" icon
  - **Apple** button with Apple icon
- Add an "or" divider between the OAuth buttons and the email form (horizontal line + "or" text, like the reference)
- Each button calls `lovable.auth.signInWithOAuth("google"|"apple", { redirect_uri: window.location.origin })`
- Buttons styled to match the modal's dark theme: `bg-[#2a2a2f] border border-white/10 rounded-xl` with provider icons

### File: `src/pages/Auth.tsx`
- Same OAuth buttons added to the standalone auth page for consistency
